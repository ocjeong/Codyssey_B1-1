
# 1. 비교 목적 및 환경

- **비교 목적:** 개별 학습자 데이터를 분석하여 퍼실리테이터에게 최적의 개입 타이밍과 맞춤형 멘토링 방안을 제안하는 데 가장 적합한 LLM 선정
- **테스트 환경:**
    - **모델 A:** OpenAI GPT-o3 (유료 / 웹 / 2026.7.1)
    - **모델 B:** Anthropic Claude 4.5 Haiku (무료 / 웹 / 2026.7.1)
    - **모델 C:** Google Gemini 3.1 Flash-Lite (무료 / 웹 / 2026.7.1)
- **※ 무료 버전 사용 시 제한점:** Claude 4.5 Sonnet 무료 버전은 사용량 한도와 단기 문맥(컨텍스트) 윈도우에 내부적인 소프트 캡이 걸려있을 수 있음.

# 2. 공통 입력 프롬프트
```당신은 PBL(Project-Based Learning) 커뮤니티 학습 환경에서 퍼실리테이터를 돕는 'AI 어시스턴트'입니다.
당신의 목표는 학습자의 데이터를 분석하여 퍼실리테이터가 언제, 어떻게 개입해야 할지 구체적이고 실용적인 행동 지침을 제안하는 것입니다.
퍼실리테이터를 존중하고 격려하는 전문가다운 톤앤매너를 유지해 주세요.

[학습자 데이터]
- 신분/전공: 재직자 / 비전공자
- 과제 수행 진도율: 35% (권장 진도율 60% 대비 지연)
- 동료평가: 요청 1회 / 참여 0회
- 시험 응시/통과 여부: 1회 응시 / 미통과(과락)
- 그룹 과제 참여 신청: 미신청 (마감 2일 전)
- 교육과정 그룹 챗 활동 정도: 눈팅만 함 (메시지 작성 0건)
- 오프라인 교육장 상태: 현재 교육장 아이맥(iMac)에 로그인하여 재석 중

위 데이터를 바탕으로 다음을 수행하세요:
1. 이 학습자의 현재 상황과 겪고 있을 예상 어려움을 데이터에 근거하여 추론하세요.
2. 퍼실리테이터가 당장 오늘 오프라인 교육장에서 취할 수 있는 구체적인 행동(멘토링 멘트 포함)을 제안하세요.
3. 분석을 마친 후, 퍼실리테이터가 이 학습자에 대해 추가로 알고 있는 정성적 정보가 있는지 반드시 '되물어(질문)' 주세요.
```

## 축별 평가 근거

### 가. 안전장치(되묻기) 이행력

- **모델 A (5점):** "3. 추가 정보 요청" 섹션을 명확히 분리하고, 업무량·학습 선호·피드백 방식·과거 경험 등 4가지 구체적 항목을 나열해 되물음. 형식적 완결성이 가장 높음.
- **모델 B (4점):** 마지막 문단에 되묻기를 포함시켰으나, 답변 흐름에 자연스럽게 녹여 넣은 형태라 "안전장치로서의 되묻기"라는 별도 스텝이 다소 약하게 드러남. 질문 항목 수도 3개로 상대적으로 적음.
- **모델 C (5점):** 되묻기를 아예 하나의 대분류 섹션(3️⃣)으로 독립시켜, 개인 정황/학습 공동체 관계/동기 관련 3개 카테고리로 세분화. 지시문의 "반드시 되물어달라"는 요구를 가장 체계적으로 이행.

### 나. 페르소나 및 톤앤매너 유지

- **모델 A (4점):** "전문가다운" 톤은 잘 지켰으나, 표(table) 중심의 딱딱한 구성이 반복되어 "존중과 격려"라는 정서적 색채는 다소 약함.
- **모델B (5점):** "든든한 동료로서 다가가라"는 구체적 접근법 제시, 시작·끝 인사에서 퍼실리테이터에 대한 격려("노고가 많으십니다")를 자연스럽게 배치. 지시문이 요구한 존중·격려·전문성 3요소를 가장 균형 있게 살림.
- **모델 C (4점):** 격려 멘트 자체는 풍부하나, 이모지(🔴🟠📊🎯 등)와 다단 구조가 과다해 "전문가다운" 절제된 톤보다는 다소 산만한 인상. 실용 정보량은 많지만 톤의 일관성은 A·B에 비해 약간 떨어짐.

### 다. 행동 제안의 구체성 및 실용성

- **모델 A (5점):** 오늘/48시간/주간 단위로 액션플랜을 시분화하고, 이후 트래킹 표(EOD/+2일/+7일)까지 제시해 "당장 오늘"과 "이후 지속 관리"를 모두 커버. 실행 가능성이 가장 높음.
- **모델 B (3점):** 멘토링 멘트와 2가지 실행 포인트는 있으나, 시간 단위 세분화나 사후 트래킹 계획이 없어 "오늘 무엇을, 언제까지, 어떻게 확인할지"에 대한 해상도가 낮음.
- **모델 C (5점):** Step1~4로 시간(분 단위)까지 명시하고, 소통 채널 확립까지 포함해 재이탈 방지책도 마련. A와 마찬가지로 즉시 실행 가능한 스크립트 수준의 구체성을 갖춤.

### 라. 근거 제시/추론의 정확성

- **모델 A (5점):** 데이터별로 "의미/원인 → 예상 어려움"을 표로 1:1 매핑하고, 이를 종합해 "고립형 학습 정체"라는 진단명으로 압축. 논리적 비약 없이 데이터에 밀착된 추론.
- **모델 B (3점):** "번아웃 직전", "골든타임"이라는 해석은 설득력 있으나, iMac 로그인 사실을 "마지막 끈을 잡고 있다는 신호"로 해석하는 부분은 다소 감정적·비약적 추론(데이터가 실제로 뒷받침하지 않는 서사적 해석)에 가까움.
- **모델 C (4점):** 지표별 의미와 심각도(🔴🟠)까지 표기해 근거를 세분화했으나, "학습 이탈의 초기 단계"라는 결론이 A의 "고립형 학습 정체"보다 다소 일반적인 진단이며, 일부 항목(채팅 눈팅=심리적 여유 부족)은 해석의 근거가 약함.
